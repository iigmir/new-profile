個人簡介

自我簡評

喜歡低調、喜歡文字、喜歡自己動手、喜歡從無到有的感覺。
從「不知道什麼是 DNS」開始，自學到今天的的網路開發者。

自傳

動機

大學畢業後，發現自己對操作電腦深感興趣，並希望能把操作 電腦轉化為賴以維生的技能，幾經考慮，最終決定參加網路工程的職業訓練。在職業訓練的過程中，我有些意外地通過了 CCNA 的認證。 也因為有了這個經驗，我大大提昇了對資訊業的興趣。

之後參與幾場面試，發覺許多企業有開發網站的需求。於是，我決定自己學習撰寫網頁語法。就算是為了生計而去從事其他領域的工作，也難以忘懷在電腦面前開發網站的興奮感受。同時在工作中，我發現自己在解決同事的手機問題上頗有成就感，使我更堅信自己要是能投身資訊業，一定能透過自身的專業，幫上更多的人解決問題。

直到後來，我發現自己已經到了「與自己的技能進步相比，更需要與人合作、以及人脈結交」的階段時，決定辭去工作，專注於尋找與人合作的經驗，並決定參加 RWD 職訓，以結交設計方面的朋友。

旅途

在職訓時，也因為熟捻網頁語法，所以常常會去幫助別人。不只 學員對我印象深刻，連老師也對我有興趣，告訴我去接觸伺服器端語言，會對生涯發展更有幫助。因此，我決定去學習後端。幾經探索，最後選擇了 Ruby 作為開發語言。

我漸漸地從一無所知，變成了開發者。之後，決定選擇以前端開發當主要工作。

第一份工作

在第一份前端開發中，我學到了使用後端 API 處理前端的方法、也學會了處理瀏覽器問題的一些小技巧。另外，我藉由同事的程式碼學會如何令 Vue 的寫法更加優雅；在處理 Laravel 的時候，無形中影響了我管理 rails 的方式。

第一份與第二份前端開發間，除了整理作品集外，還寫了小作品如網路爬蟲、任務執行器等。

第二份工作

第二份工作的技術更具挑戰性：我真正參與、並開發產品級的 SPA(Single Page Application) 專案。這大大增進了我技術上的思考；此外，也由於開發者的增多、以及地位的交換，我也學會了一些溝通與分工上的技巧。

學習與貢獻是相互形成的。技術與溝通的增進，也會對公司的開發速度有所幫助：同事對我在熱心和技術上的評價不錯；而專案的開發進度，通常能比預估進度快上數週。

未來

若有幸與公司共事，我除了希望專注前端開發趨勢外，也會嘗試理解伺服器端語言，最終走向網站全端開發。
